EPP Documentation

Everything you need to write, parse, and render EPP documents. Covers both v0.1 and the new v0.2 additions.

Introduction

What is EPP?

EPP (Easy Paper Package) is a lightweight, plain-text document format. A .epp file is a sequence of UTF-8 lines — no binary blobs, no embedded resources, nothing that isn't readable in a text editor.

EPP is designed for the simple case: a document that fits on A4 paper, has headings, paragraphs, lists, and maybe a table or two. Think of it as the sweet spot between a plain .txt file and a full PDF — structured enough to print cleanly, simple enough to last forever.

  • Plain UTF-8 text — open in any editor
  • Line-oriented and command-based — trivial to parse
  • No internet, no binaries, no runtime dependencies
  • Designed to be printed on A4 paper
EPP is not a replacement for Markdown, DOCX, HTML, or PDF. It fills a different niche: long-term preservation of simple printable documents.

File format

Version header

Every EPP file should start with a version declaration enclosed in percent signs:

%epp=0.2%

This tells the renderer which spec the file was written against. The header is optional but strongly recommended.

Comments

A semicolon starts a comment. Everything from ; to end-of-line is ignored.

; This entire line is a comment
@text "Hello"  ; inline comment — also ignored

Syntax

General command form

Most EPP commands follow this pattern:

@command "text" {attributes}

The @ prefix identifies a command. The text argument is always in double quotes. The attribute block in curly braces is optional.

Some commands take no text argument at all (e.g. @space, @line, @newpage). Some take a bare word instead of a quoted string (e.g. @page 1, @label mymark).

Attribute block

Attributes are a comma-separated list inside { }. Key-value pairs use =, boolean flags are bare words:

{bold, italic, align=center, color=blue, type=check}

Commands

Every command that exists in EPP v0.2 is listed below. Commands marked new were added in v0.2.

@title
Sets the document title (used in the viewer titlebar).
@title "My Document"
@heading
A section heading. Supports bold, italic, underline, align, color.
@heading "Section" {bold, align=center}
@text
A paragraph of body text.
@text "Body copy here." {align=left}
@bullet
A list item. Use type= to choose the glyph style.
@bullet "Item" {type=check}
@code
Monospace code block. Use [newline] for line breaks.
@code "print('hi')[newline]exit()"
@space
Inserts a vertical gap between blocks.
@space
@label
Named bookmark for navigation. Appears as a tab in the viewer.
@label intro_end
@page
Sets the current page's number/label shown in the footer.
@page 1
@newpage
Ends the current page and starts a new one.
@newpage
@meta new
Document metadata key-value pair (title, author, version…).
@meta author="Your Name"
@header new
Text printed in a ruled band at the top of every page.
@header "My Document"
@footer new
Text printed at the bottom of every page. Use [page] for the page number.
@footer "Page [page]"
@quote new
A blockquote — italic, amber left border.
@quote "Keep it simple."
@line new
A horizontal rule divider.
@line
@table new
Starts a table. Pipe-separated column headers.
@table "Name|Role|Age"
@row new
Adds a data row to the last table. Must follow a @table.
@row "Alice|Dev|25"

Attributes

Attributes modify how a block is rendered. They go inside { } after the text.

AttributeValuesCommandsNotes
boldflagheading, textSets font-weight 700
italicflagheading, textNew in v0.2

alignleft · center · rightheading, textHorizontal alignment
colorblack · red · blue · green · gray · amber · purpleheading, text, bulletRenderer hint
typedisc · number · check · arrow · starbulletNew in v0.2 (disc = default •)
Numbered lists: consecutive @bullet commands with {type=number} auto-increment. The counter resets when a non-bullet block appears.

Pages

Documents are divided into pages using @newpage. Each page can have an optional number set with @page.

@page 1
@heading "Cover"
@newpage
@page 2
@heading "Chapter One"

The @header and @footer commands (v0.2) are document-level — declare them once before any @page and they appear on every page automatically.

Escape sequences

Inside quoted strings, these special forms are recognized:

EscapeProduces
\"A literal double-quote character
\\A literal backslash
\[A literal [ (otherwise starts a token)
\]A literal ]
[newline]A line break inside the rendered text
[page]Current page number (in @footer only)

Full example

A complete v0.2 document using all major features:

%epp=0.2%
; Document metadata
@meta title="Project Report"
@meta author="SD"

; These repeat on every page
@header "Project Report"
@footer "Page [page]"

@page 1
@title "Project Report"
@heading "Introduction" {bold, align=center}
@text "This document covers the Q3 project summary."
@space
@quote "Simplicity is the ultimate sophistication."
@line
@heading "Steps completed"
@bullet "Research phase" {type=check}
@bullet "Design review" {type=check}
@bullet "Implementation" {type=arrow}

@newpage
@page 2
@heading "Team"
@table "Name|Role"
@row "Alice|Lead"
@row "Bob|Design"
@space
@code "$ make build[newline]$ make test"
Try it: paste this into the Editor using the Import button.

What's new in v0.2

v0.2 adds document-level metadata, page chrome, richer blocks, and expanded bullet styles. All v0.1 files are valid v0.2 — the version header is the only required change.

  • @meta key="value" — document metadata (title, author, version, …)
  • @header "text" — text in a ruled band at the top of every page
  • @footer "Page [page]" — footer on every page; [page] expands to the current page number
  • @quote "text" — blockquote with italic style and amber left border
  • @line — horizontal rule
  • @table "Col1|Col2" + @row "v1|v2" — simple tables
  • {italic} and {underline} attributes on heading and text
  • {type=number|disc|check|arrow|star} on bullet
